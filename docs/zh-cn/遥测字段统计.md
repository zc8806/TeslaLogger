## 四、处理机制说明

### 1. 字段路由与分类处理

```
TelemetryParser.handleMessage(json)
  │
  ├─ 电池字段 → InsertBatteryTable() → battery表
  ├─ 巡航字段 → InsertCruiseStateTable() → cruisestate表
  ├─ 状态字段 → handleStatemachine() → 控制Driving/acCharging/dcCharging标志
  ├─ 位置字段 → InsertLocation() → pos表（仅行驶时）
  ├─ 充电字段 → InsertCharging() → charging表（仅充电时）
  └─ 其他字段 → InsertStates() → pos/TPMS表或内存状态
```

### 2. 写入条件控制

```csharp
// 代码位置：TelemetryParser.cs

// 电池表写入条件
if (PackVoltage.HasValue && PackCurrent.HasValue)
    InsertBatteryTable();

// 位置表写入条件
if (Driving == true && Location.HasValue)
    InsertLocation();

// 充电表写入条件
if (IsCharging == true && 有充电字段)
    InsertCharging();
```

## 五、总结

**当前项目实际接收并处理的特斯拉遥测字段为 46 个**，主要覆盖核心车辆状态、电池信息、充电状态、驾驶辅助和基本信息。未处理的字段主要是高级 HVAC 控制、详细门窗状态和高级软件更新信息。

**配置文件与代码的差异说明**：
配置文件中的字段列表是 TeslaLogger 官方推荐的完整字段集合，而代码中实际处理的字段是经过筛选的核心字段，主要考虑：
1. 数据重要性
2. 存储成本
3. 处理复杂度
4. 用户需求

如果需要支持更多字段，可以在 `TelemetryParser.cs` 中添加相应的字段处理逻辑。
